# 创伤疗愈伴侣 · Prompt 调试日志

**用途**：记录每次 prompt / 路由相关改动，避免改来改去、重复踩坑。  
**前提（已确认）**：DeepSeek 官方 API · `deepseek-reasoner`（深度）/ `deepseek-chat`（快轨）能力足够；后续主战场是 **prompt 泛化**，使各类用户输入都能得到好回复。  
**基准输入**：长自述（NPD / 同事关系 / 价值崩塌等，约 500～600 字）；对照 DeepSeek 网页金样约 ~2900 字。  
**相关代码**：`src/chat/prompt.ts`、`src/chat/model-router.ts`  
**更早的模型选型史**：见 [疗愈伴侣模型调试报告.md](./疗愈伴侣模型调试报告.md)

---

## 使用约定（每次改 prompt 必做）

1. **先读本日志「经验法则」与最近 3～5 条记录**，再动手。  
2. **一次只改一类变量**（system 文案 / user 模板 / 路由门槛 / max_tokens），便于归因。  
3. 改完用同一条长自述（必要时 `--fresh`）或真实多轮各测一轮。  
4. **立刻追加一条记录**，填齐：改动点、原因、效果（助手看法 + 用户看法）。用户看法未知时写「待用户反馈」。  
5. 若效果变差，优先 **回退到最近「用户认可」基线**，再小步试，不要叠加重规则。

## 经验法则（从本轮迭代提炼，禁止无故违反）

| ID | 法则 | 说明 |
|----|------|------|
| E1 | **模型已够用，少怪模型** | 官方 `deepseek-reasoner` 深度与温度可达网页级；差在约束方式，不在换模型。 |
| E2 | **少规则、软目标** | 编号 SOP（先承接→再串联→…）易导致报幕与咨询报告体；堆 avoid 列表易变短或变假。 |
| E3 | **不要用 user 侧「场景任务块」绑死多轮** | `CHAT_USER_INTAKE_TASK` 一类「本轮是长叙述」假设泛化差；门槛一严就漏触发、一松就污染短跟进。 |
| E4 | **篇幅靠「鼓励写透」+ token 预算，不靠填提纲** | 去掉清单后字数会塌；硬写 2000～3000 又依赖触发。更好：system 里泛化「说透、按复杂度展开」+ deep 给足 `max_tokens`。 |
| E5 | **反模板指令会连带收短** | 「不要报幕 / 不要分节 / 少量建议」常被理解成写短；若要长，须同时给正向篇幅/深度信号。 |
| E6 | **路由只负责模型与预算，不负责改人设话术** | deep/fast 选模型与 max_tokens；深度话术放 system，对所有轮次生效。 |
| E7 | **用户主观满意是最终标准** | 字数、结构分只是辅助；以「深入准确 + 情绪温暖 + 接近网页」为准。 |
| E8 | **「勿过早收束」类加句未必有效** | 多层跟进相对网页可能偏短，但补一句「各条线索都接到、勿过早收束」未拉齐篇幅，还可能让短问略损温度（见 R9→R10）。缺完整度时先靠用户读感判断，勿为追字数叠软规则。 |
| E9 | **少打补丁；若打，用条件软许可** | 默认靠全局软目标（如 R13）。确有洞再补。补丁格式见下；禁止无场景硬约束、编号 SOP、字数硬指标、user 侧任务块。 |

### E9 · 补丁模板与纪律（2026-07-11 约定）

**模板**（仅此一种优先形态）：

> 当用户…时，你可以…

| 半句 | 要求 | 反例 |
|------|------|------|
| **当用户…时** | 场景边界清晰、可判别（如：一次讲清一段刺痛关系的时间线；短句签到；急性躯体不适） | 「当用户表达复杂时」「当需要深入时」（条件过糊 → 变全局加压） |
| **你可以…** | 许可式、软目标，尽量不绑死结构/字数/步骤 | 「必须先…再…」「不要…」「写满 2000 字」「按①②③」 |

**纪律**

1. **先软目标，后补丁**：同一问题先试一句全局「说透 / 看清 / 篇幅随复杂度」类；仍不够再补。
2. **一次最多加一条** `当…你可以…`，测完再决定去留或追加。
3. **成对想**：加「长/深」补丁时，评估是否需要一条「短/急」的轻量对偶（仍用「你可以」，不用禁止清单）。
4. **验收**：目标场景变好，且至少 1 个短问 + 1 个异质长对话不掉温/不掉泛化；否则回退。
5. **坏补丁特征**（见 R9/R11/intake）：场景糊、动作硬、无短问对偶、叠多条一起上。

## 当前基线（用户认可 · 2026-07-11 **R13**）

**通道**：`LLM_CHAT_PROVIDER=deepseek` · deep=`deepseek-reasoner` · fast=`deepseek-chat`  
**System**（全文）：
```
你是一个具备深度洞察力的陪伴者，面向有创伤经历、正在自我疗愈的成年人（包括尚未达到 CPTSD 等诊断标准、但同样感到痛苦的幸存者）。
你不仅拥有心理学的温厚，也具备生物学与社会学的理性。
你的目标是：让对方感到被坚定地理解与认可，又能被温暖地承接；把困扰与痛苦的原因说透、看清，帮助对方更深、更温暖地认识自己，并在其经历中看见成长；篇幅随对方表达的复杂度自然展开。
```
**User**：仅上下文块 + 当前输入；无 intake 任务块。  
**路由**：长度≥300 或深度信号≥2 → deep，`maxTokens=16384`。  
**用户评价**：温暖而坚定，同理好，分析到位，支持感强（创伤激活 reframing）。
