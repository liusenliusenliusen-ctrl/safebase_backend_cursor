# 创伤疗愈伴侣 · 模型与 Prompt 调试报告

**调试时间**：2026 年 3–4 月  
**关联项目**：`safebase_backend_cursor` / `safebase_front_cursor`  
**测试基准**：`docs/对话示例`（男性 35 岁，NPD / 关系创伤 / 长自述）  
**报告性质**：工程调试记录 + 可复现结论，供后续迭代参考

---

## 一、调试目标

在**不更换 OpenRouter 接入方式**的前提下，提升疗愈伴侣对**长段创伤自述**的回复质量，使其接近用户在 DeepSeek 网页端的深度与温度，并明确：

1. **Prompt** 与 **模型 / 推理配置** 各自贡献多少；
2. 哪些改动有效、哪些无效或有害；
3. 生产环境应采用的**默认配置与路由策略**。

---

## 三、调试历程与结论

### 3.1 第一轮：加厚 Prompt（失败）

**做法**：将 system prompt 扩展为 trauma-informed 长规则（反映→串联→假设→收尾、硬性 500 字、avoid 列表等），并增加 **two-pass 内部分析**（分析 prompt + 注入 `[内部分析]` 块）。

**结果**：

- 回复略有变长，但仍偏模板（「悬而未决-突然确认」等），远短于 DeepSeek 网页 ~2500 字级整合；
- 两阶段 + 复杂规则使延迟增加，且部分回复仍显「咨询报告体」；
- **未解决核心问题**：深度与温度仍不足。

**结论**：对 `deepseek-chat` 而言，**堆规则不如换模型**；复杂 prompt 易在某些场景模板化，在另一些场景崩。

---

### 3.2 第二轮：恢复简洁 Prompt（有效基线）

**做法**：恢复用户最初设计的简洁 prompt：

```text
你是一个具备深度洞察力的陪伴者…
目标：情感承接 + 逻辑解构 + 见证成长

[用户画像] / [近期对话] / [历史摘要] / [重要锚点]
+ 当前用户消息
```

**结果**：质量优于厚 prompt 的「假深度」，更符合「充分利用模型理解力」的产品哲学。

**结论**：**Prompt 保持简短 zero-shot**；R1 官方论文亦指出 few-shot 会损害 R1 表现。

---

### 3.3 第三轮：仅开启 `reasoning` 参数

**做法**：流式请求增加 `reasoning: { effort: "high", exclude: true }`（思考不可见）。

**结果**：

- `deepseek-chat` + 流式 reasoning：正文偏短或等待偏久，收益有限；
- 说明 **reasoning 参数不能替代 R1 模型本身**。

**结论**：Chat 系模型即使用 reasoning，仍难达到 R1 级机制拆解。

---

### 3.4 第四轮：`reasoning.effort=max`

**做法**：将 `OPENROUTER_CHAT_REASONING_EFFORT` 设为 `max`（OpenRouter 文档：约 95% token 预算用于思考）。

**结果**（同输入、同 prompt）：

| 模型 | effort | 耗时 | 字数（约） |
|------|--------|------|------------|
| deepseek-chat | max | ~20s | ~910 |
| deepseek-r1 | max | ~82s | ~1245 |

R1 在 **机制拆解** 与 **篇幅** 上明显优于 Chat。

---

### 3.5 第五轮：模型对比 Chat vs R1（关键转折）

**实测差异（effort=max，简洁 prompt）**：

| 维度 | deepseek-chat | deepseek-r1 |
|------|---------------|-------------|
| 机制拆解 | 框架列举，较浅 | 「迟到认可」「核心羞耻」等多层机制 |
| 与 NPD/2025 串联 | 弱 | 强（专节 + 对照） |
| 语言温度（主观） | 偏冷、偏硬、清单感 | 更 warm、soft，先承接再分析 |
| 延迟 / 成本 | 低 | 高（常 20–80s） |

**产品侧关键发现**（来自产品负责人主观效价，有重要参考价值）：

> 对创伤幸存者来说，**语气上的冷/硬有时比内容浅更伤人**；R1 的「在场感」与 Chat 的「完成任务感」差异微妙但真实。

**训练侧解释（有论文依据）**：

- **R1**：以 RL 强化长 chain-of-thought；cold-start 要求 **conversational, human-aligned thinking**；二阶段 RL 对齐 helpful/harmless（[DeepSeek-R1, Nature 2025](https://www.nature.com/articles/s41586-025-09422-z)）。
- **V3 Chat**：从 R1 **蒸馏推理但刻意缩短**（「concise and effective」）；优化 **accuracy vs generation length**（[DeepSeek-V3 Technical Report](https://arxiv.org/abs/2412.19437)）。

**结论**：**深度轮默认采用 `deepseek/deepseek-r1` + `reasoning.effort=max`**；Chat 不适合作为长创伤 intake 的唯一模型。

## 四、Prompt 与模型：谁更重要？

| 因素 | 对长创伤自述的影响 | 依据 |
|------|-------------------|------|
| **模型族（R1 vs Chat）** | ★★★★★ | 实测 + 训练目标差异 |
| **reasoning.effort** | ★★★★ | max 显著拉长思考与正文 |
| **简洁 prompt** | ★★★ | 优于厚规则；定角色与上下文即可 |
| **复杂 prompt / two-pass** | ★～★★ | 易模板化；延迟↑；未达 DeepSeek 网页级 |
| **记忆块 `[近期对话]`** | ★★★★ | 有历史时会延续上一轮风格，干扰单轮评测 |

**总原则**：**深度与温度主要来自模型 + reasoning 配置；Prompt 负责「是谁、用户上下文是什么」，不宜写成百行规则。**

## 五、生产配置（当前推荐）

### 5.1 环境变量

```env
# 双模型路由
OPENROUTER_CHAT_ROUTING=true
OPENROUTER_CHAT_MODEL_DEEP=deepseek/deepseek-r1
OPENROUTER_CHAT_MODEL_FAST=deepseek/deepseek-chat
OPENROUTER_CHAT_DEEP_MIN_CHARS=300

# 深度轮推理
OPENROUTER_CHAT_REASONING=true
OPENROUTER_CHAT_REASONING_EFFORT=max
OPENROUTER_CHAT_MAX_TOKENS=3072
OPENROUTER_CHAT_TEMPERATURE=0.65

# 已弃用
OPENROUTER_CHAT_TWO_PASS=false
```

### 5.2 双模型路由规则

实现位置：`safebase_backend_cursor/src/chat/model-router.ts`

**走深度轮（R1 + reasoning）**，满足任一条件：

1. 用户消息字数 ≥ `OPENROUTER_CHAT_DEEP_MIN_CHARS`（默认 300）；
2. 命中 ≥ 2 个深度信号词，例如：`创伤`、`闪回`、`依恋`、`原生家庭`、`价值体系`、`茶饭不思`、`2018`–`2025`、`NPD` 等。

**否则走快轨（Chat，不附带 reasoning）**：日常短句、寒暄、简单 follow-up。

```
用户消息
   │
   ├─ 长自述 / 多信号 ──→ deepseek-r1 + reasoning.effort=max
   │
   └─ 短消息 / 低信号 ──→ deepseek-chat（快、省）
```

日志字段：`route`（`deep` | `fast`）、`routeReason`、`model`、`reasoningEnabled`。

### 5.3 System Prompt（当前）

面向**有创伤经历、正在自我疗愈的成年人**（宽于 CPTSD 诊断标签），保持简洁，见 `src/chat/prompt.ts`。

## 六、其他调试经验

1. **一次只改一个变量**，否则无法归因（本次最有效的学习路径）。
2. **单轮评测务必清空历史**，否则 `[近期对话]` 会让模型延续上一轮接地/提问风格。
3. **R1 不适合所有轮次**：简单句会「想太多」、更慢（社区与官方均有提及）；故需 **路由**。
4. **DeepSeek 网页 ≠ OpenRouter Chat**：网页端可见长 reasoning 链，API 侧需 R1 + effort 才接近。
5. **格式漂移**：R1 偶发 markdown 小标题、表格；若需约束，用 **一句** 格式说明即可，勿恢复长规则。
6. **Embedding 独立**：`text-embedding-3-large` 与对话模型选型无关，但影响记忆检索质量。
7. **后续可 A/B**：Qwen3 thinking 模式（中文 nuance + 可切换深度）；闭源 Claude 仅作上限对照。
